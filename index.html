Gameplay highlights

These are the parts of the project that show the work most clearly: buying, learning, navigating, gearing up and moving through the world.

Dragon commerce

Dragons are bought directly from an in-world entity shop using coins, with the full flow handled inside the add-on.

  • The shop uses an entity interaction instead of a regular NPC trade window.
  • Coin cost is shown in-world and tied to the selected dragon.
  • Successful purchases grant the matching dragon egg.

Gear and combat loadouts

The project includes custom weapons, armor sets and support items tied to combat and progression.

  • Custom swords and bows connect directly to scripted combat behavior.
  • Armor sets matter as gameplay, not just as visuals.
  • The item art keeps the add-on readable while still giving it its own style.

Guides, potions and NPC shops

The Dragon Guide helps explain mechanics, consumables, cosmetics and healing, and potions can also be bought through an NPC merchant flow.

  • The guide explains dragon mechanics, potions, cosmetics and bandages.
  • The potion pages make heavier systems easier to learn in-game.
  • The NPC merchant gives the project a second type of shop flow besides the dragon store.

GPS and player direction

Players can choose destinations through the General Map, follow actionbar guidance while traveling and get confirmation when they arrive.

  • Trackable destinations include shops, quests, the arena and other key locations.
  • The system connects menu selection to actual movement through the world.
  • Arrival feedback makes the navigation loop feel complete.

Custom Spells

A collection of custom magical abilities built entirely via Script API, featuring area-of-effect (AoE) calculations, timing loops, and status effect management.

Rain of Fire

Fire falls from the sky into a targeted area. Enemies caught inside the zone take continuous burn damage and are slowed for 5 seconds.

Experimental System

Battle Challenges: Custom JSON UI & Data-driven mechanics.

I built this experimental system to push the limits of Bedrock's JSON UI. My goal was to create a custom interface that felt completely unique, breaking away from the standard vanilla look while maintaining a tight integration with the Script API.

Custom UI & Workflow

This project was a deep dive into custom component design and automation, from the pixel art to the TSX generators.

JSON UI Handmade Assets Regolith Generators Data-driven Script API Pixel Art
  • HUD Integration: I modified the vanilla hud_screen to support live objective tracking, giving players immediate feedback on their progress without opening a menu.
  • Handmade Assets: I designed every button and frame from scratch, ensuring the pixel art style stayed consistent across the entire custom interface.
  • Regolith Automation: Using generators, I built a data-driven pipeline that makes adding new challenges as simple as editing a config file.
  • Experimental Logic: A prototype for complex challenge tracking and achievement rewards built as a technical exercise.
